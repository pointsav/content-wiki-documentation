The PointSav Location Intelligence Platform is a co-location cluster analysis system that identifies and ranks commercially defensible retail real estate nodes. It ingests publicly licensed store location records from OpenStreetMap and the Overture Maps Foundation, applies a named-anchor proximity model to score each location, and renders results as an interactive layered map. All data, tiles, and algorithms are owned by the customer and run on customer infrastructure — no ongoing per-request fee is required. The first production deployment serves `gis.woodfinegroup.com`, built in 2026.

## Named-Anchor Model

The platform applies the Named-Anchor Model to score retail locations. When independent, capital-intensive operators converge on the same geographic node, that convergence validates the trade corridor. The model categorises these operators by commercial function and catchment radius:

**Primary Target** — the foundational requirement for any evaluated node. In North America, the Walmart Supercentre. In European markets, IKEA serves as the operating primary target under the current analysis configuration; this reflects the chain's role as the dominant traffic anchor in European retail corridors.

**Secondary anchors** — large-format complementary operators evaluated within a 1.0 km to 3.0 km radius of the Primary Target: hardware operators (Home Depot, Lowe's, Leroy Merlin) and warehouse-club formats (Costco, Sam's Club, Makro).

**Tertiary anchors** — civic and institutional infrastructure evaluated within a 5.0 km radius of the cluster centroid: major hospitals, medical centres, universities, and colleges.

Full methodology — including catchment radii, named-anchor role definitions, and regional operator equivalents — is documented in [[co-location-methodology]].

## V2 Scoring Model

Sites are evaluated on a 0–1000 scoring scale and classified into three quality tiers based on score_final:

| Rating | Tier | Name | Threshold | Commercial Validation |
|--------|------|------|-----------|----------------------|
| ★★★ | T3 | Apex | score_final ≥ 700 | Absolute commercial convergence; small fraction of all evaluated locations. |
| ★★ | T2 | Hub | score_final ≥ 450 | Commercial pairing validated. |
| ★ | T1 | Valid | score_final ≥ 150 | Anchor + at least one secondary confirmed. |

A country saturation guard prevents grade inflation: if T3 sites in any ISO country would exceed 5% of all evaluated locations, the T3 threshold rises in 25-point increments until the proportion falls below the ceiling.

## Three-layer map architecture

**Layer 1 — Base locations.** All individual store coordinates sourced from OpenStreetMap via the Overpass API, coloured by brand family. Available as a map toggle; not the default view.

**Layer 2 — Co-location clusters.** The primary working view. Each cluster marker encodes quality tier by size and colour saturation — T3 Apex clusters are visually dominant. Selecting a cluster opens an information panel showing city, national rank, secondary anchors present, and service-places within 5 km.

**Layer 3 — Catchment radius.** A buffer around each cluster centroid defining the scope for trade-area analysis. Default radius is 75 km; dense urban markets use 25–30 km; sparse or rural markets extend to 150 km. This boundary identifies the geographic origin pool relevant to each cluster's trade area.

## Data sovereignty

All components of the platform are customer-owned:

- Retail location records — flat-file JSONL, stored in the platform's service-business data layer
- Institutional anchor records — Overture Maps Foundation data, stored in the platform's service-places layer
- Map tiles — PMTiles format, generated by Tippecanoe, served by nginx with no external CDN dependency at runtime
- Scoring algorithm — open Python pipeline that reads from the data layer and writes cluster records to the deployment

If the GIS gateway becomes unavailable, all data persists in the Totebox Archive and can be restored to a replacement gateway instance. The underlying flat-file architecture is described in [[location-intelligence-substrate]].
